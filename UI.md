# SYS.INT Design System — Style Specification

## Core Aesthetic

**Retro-industrial terminal meets precision engineering.** Think: a military-grade operations console from 2045, built by systems engineers who hate decoration but love function. Every element should feel like it was generated by a deterministic process, not designed by a human.

## Motion & Animation

```css
/* Minimal. Purposeful. No flourishes. */

/* Uptime counter — monospace ticker */
@keyframes tick { to { content: attr(data-next); } }

/* Cursor blink on active inputs */
@keyframes caret-blink { 50% { opacity: 0; } }

/* Subtle entrance: fade + 4px translate up, 300ms */
@keyframes enter {
  from { opacity: 0; transform: translateY(4px); }
  to   { opacity: 1; transform: translateY(0); }
}

/* NO: bounces, springs, elastic, elaborate sequences */
/* YES: instant state changes, subtle opacity, terminal-style reveals */
```

## Design Principles

1. **Monospace everything** — no humanist or geometric sans-serifs anywhere
2. **Dotted grid background** — `radial-gradient` 20px spacing, warm off-white base `#E8E4DC`
3. **Square corners** — only pill exception for node labels
4. **Accent = `#D0F501`** — used sparingly: live indicators, key stats, accent lines, connector dots
5. **All UI labels are ALL CAPS + tracked** — treat every label like a terminal command
6. **Dark panels are true dark** (`#1C1C14`), not dark gray
7. **Borders are structural**, not decorative — every border means something
8. **Section coordinates** in corners (bottom-left section name, bottom-right section number)
9. **Zero drop shadows** — depth comes from borders and background contrast only
10. **Bitmap/dot-matrix display font** for hero headlines only — everything else is clean monospace
